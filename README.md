CS 250 Software Development LifeCycle

**How do I interpret user needs and implement them into a program? How does creating user stories help with this?**

I interpret user needs by breaking them down into smaller, manageable pieces. Instead of trying to build everything at once, I focus on understanding what the user wants the program to accomplish and then identify the features needed to achieve that goal. When developing a program, I usually start with the core functionality first. Once the program is working correctly, I can focus on improving the design, appearance, and overall user experience.

Creating user stories helps because they provide a clear understanding of what the user expects from the software. User stories break larger requirements into smaller, more specific goals that are easier to develop and test. They also help keep the team focused on delivering features that provide value to the user rather than making assumptions about what should be built.

**How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?**

My approach to developing programs is to divide the project into smaller tasks so I do not become overwhelmed. I like to think of these tasks as milestones that help track progress and make larger projects more manageable. Completing smaller objectives along the way also helps maintain momentum throughout the development process.

One Agile practice I would like to continue using is planning work in small, achievable increments. Agile emphasizes flexibility and continuous improvement, which helps teams adapt when requirements change. I also appreciate the focus on prioritizing features that provide the most value to users first. Rather than spending too much time on cosmetic improvements early in development, I believe it is more important to ensure the program functions correctly and meets user needs.

**What does it mean to be a good team member in software development?**

To me, being a good team member means communicating effectively and working collaboratively with others. Strong communication helps ensure everyone understands project goals, expectations, and any challenges that arise during development. It is not enough to simply communicate; information must be shared clearly so the entire team can stay aligned and make informed decisions.

A good team member is also willing to listen to feedback, support others when needed, and contribute to the team's success. When team members communicate openly and work together toward a common goal, projects are more likely to be completed efficiently and successfully.
